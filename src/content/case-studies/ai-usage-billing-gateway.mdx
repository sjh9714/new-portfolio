## TL;DR

AI API 사용량을 멀티테넌트 단위로 수집하고, API key 보안, usage idempotency, invoice 중복 생성, webhook duplicate/conflict, append-only ledger invariant를 검증한 과금 백엔드 사례입니다.

## Problem

- API key 원문이 DB에 저장되면 유출 시 즉시 악용될 수 있습니다.
- timeout/retry가 있는 사용량 수집 API는 같은 usage event를 여러 번 받을 수 있습니다.
- 결제 provider webhook은 중복 delivery가 정상 동작이므로 duplicate와 conflict를 구분해야 합니다.
- ledger는 수정 가능한 잔액 컬럼만으로는 감사와 정산 근거를 설명하기 어렵습니다.

## Design Decisions

- API key는 최초 발급 시에만 원문을 반환하고 DB에는 prefix와 hash만 저장합니다.
- usage event와 webhook은 idempotency key, providerEventId, payload hash로 중복과 충돌을 구분합니다.
- ledger는 append-only invariant를 둬 debit/credit balance, 단일 currency, 양수 amount 조건을 검증합니다.
- mixed usage benchmark, scheduler/batch, dashboard, SLO는 아직 Pending으로 표시해 실무 준비도를 부풀리지 않습니다.
